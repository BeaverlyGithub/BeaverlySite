Instructions and Comfort

How to create, pause, and revoke instructions

This page covers the instruction lifecycle and the comfort controls that define your automation boundaries.

Where to find this

Instruction setup and comfort controls are available from the instruction workflow in Chilla. In many builds, controls appear in setup and instruction detail views.

Verification note: Field names and button labels may vary by release. Confirm exact UI strings before training materials are finalized.

What you can do here

  • Create an instruction with selected behavior and playground context.
  • Set per-position comfort boundaries.
  • Add optional monthly comfort boundaries (where available).
  • Activate, pause, edit, or revoke existing instructions.
  • Review status labels and recent lifecycle updates.

Create an instruction

  1. Open instruction setup. Confirm active environment selection.
  2. Select behavior and playground(s). Keep scope clear and intentional.
  3. Set comfort values. Review implications before activation.
  4. Review summary. Check key details one more time.
  5. Activate instruction. Confirm consent when prompted.

Pause or revoke an instruction

Pause

Use when you want to temporarily stop automation while keeping your setup.

  • Instruction can usually be resumed later.
  • History remains available in Flow.

Revoke

Use when you want to fully stop the instruction lifecycle.

  • Revocation often requires explicit confirmation.
  • You can create a new instruction afterward if needed.

Instruction FAQ

Can I change comfort after activation?

In most builds, yes. Apply updates from instruction controls and confirm resulting status transitions in Flow.

Why is my instruction stuck in verifying?

Check environment connection status, account readiness, and recent warnings in Flow. Then follow troubleshooting steps if needed.

Is comfort financial advice?

No. Comfort is your own user-defined boundary setting. Chilla enforces what you configure.
