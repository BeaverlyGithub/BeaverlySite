Primary source files for support truth

  • Blce/dashboard.html and Blce/js/dashboard_js/dashboard-main.js for dashboard cards and click paths.
  • Blce/mandate-settings.html, Blce/mandate-wizard.html, and Blce/js/mandate-settings.js for instruction lifecycle controls.
  • Blce/js/decision-flow.js for Flow statuses, filters, replay scopes, and skip-reason wording.
  • Blce/usage.html and Blce/js/usage.js for capacity labels and reset behavior.
  • Blce/broker-settings.html and Blce/js/broker-settings.js for environment state labels.
  • BeaverlySite/pricing.html and Blce/upgrade.html for public/in-app tier displays.

UI location verification

  • Dashboard cards: Not connected yet, Instructions, Flow, Active Playgrounds.
  • Flow chips: All, Entry, Exit, Skip, Warn, Live.
  • Instruction actions: Add, Update, Pause, Revoke.
  • Environment status labels: Automated, Connected, Disconnected, Active, Ready, Inactive.
  • Usage labels: Automation Capacity Used, Capacity Available, Daily Available.

Plan and billing verification

  • Current plan names and public descriptions.
  • Environment connection limits by tier.
  • Current pricing and cadence details.
  • Upgrade/downgrade timing behavior.
  • Refund and billing dispute handling policy.

Status and support workflow verification

  • Current status labels in production UI.
  • Expected lifecycle transitions for active instructions.
  • Which warnings appear in Flow for common failure cases.
  • Support contact mailbox ownership and response expectations.

Disclosure safety verification

  • No proprietary formulas or private decision internals disclosed.
  • No advisory language or guarantee phrasing.
  • Terminology remains user-facing: behavior, comfort, environment, instruction, Flow.
  • Non-custodial framing appears where relevant.
