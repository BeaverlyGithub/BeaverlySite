Troubleshooting

What to check when something is not working

Use this page for fast diagnostics before escalation. Start with status, environment readiness, and Flow timestamps.

Article type: Troubleshooting Last reviewed: March 12, 2026

Universal first checks

  1. Check instruction status. Active, paused, revoked, or verifying.
  2. Check environment status. Connected and ready.
  3. Check Flow. Recent warnings, skips, or readiness notes.
  4. Check usage. Daily capacity and remaining room.
  5. Check plan scope. Confirm limits match your current setup.

Issue: Login failed

  • Confirm email and password match account records.
  • Retry after refreshing session.
  • Check inbox for verification requirements.

Escalate when: repeated failures continue after credential reset.

Issue: Environment connection failed

  • Re-check account credentials and authorization scope.
  • Confirm target account is active on provider side.
  • Retry connection and watch verification status transitions.
  • Use Flow or status panel notes for failure context.

Issue: Behavior will not activate

  • Confirm behavior and playground compatibility.
  • Confirm environment is connected and selected as active.
  • Review comfort boundaries for invalid combinations.
  • Check plan and capacity state if activation is blocked.

Issue: No activity appears in Flow

  • Confirm instruction is active, not paused or revoked.
  • Check if status is monitoring or verifying.
  • Confirm environment readiness.
  • Check usage and remaining capacity.
  • Review recent warnings for step-aside context.

Issue: Capacity looks low

  • Review number of active instructions and playground coverage.
  • Check whether a high-activity window increased cycle usage.
  • Compare usage trend across recent days.
  • Consider plan change if usage pattern is now steady at higher levels.

Issue: Upgrade not reflected yet

  • Refresh session and return to plan screen.
  • Confirm billing event completed successfully.
  • Check whether cycle boundary timing applies to your change type.

Issue: Status stuck in verifying or monitoring

  • Check environment health and connection continuity.
  • Review Flow for any persistent warnings.
  • Confirm instruction scope and comfort values are valid.
  • If unchanged for an unusual duration, escalate with timestamps.

Before contacting support

  • Account email used for sign-in.
  • Instruction ID or clear instruction name.
  • Environment name and current status.
  • Recent Flow timestamps and event types.
  • Short summary: expected vs observed behavior.
